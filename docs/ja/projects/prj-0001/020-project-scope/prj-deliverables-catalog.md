# 成果物カタログ

Project Deliverables Catalog

本ドキュメントは、SpecDojo Handbook プロジェクトで作成する成果物の一覧とその説明をまとめたカタログです。各成果物の目的・内容・配置先・派生関係を定義します。

## 1. プロジェクト管理

プロジェクト運営・管理に関する成果物です。`<name>.md` もしくは、`<name>.yaml` 形式で作成し、配置先に保存します。

- ドメイン: `project`

- 配置先: `docs/ja/projects/prj-0001/010-project-overview`

| `prj-charter`              | プロジェクト憲章       | プロジェクトの正式な認可と権限委譲を文書化 |

## 2. エージェントカスタマイズ

成果物の自動生成を支援するために VS Code / GitHub Copilot エージェントへ適用するカスタマイズファイルです。

- ドメイン: `agent-customization`
- 配置先: `.github/`

### 2.1. Instructions

VS Code Copilot エージェントが自動参照するルール指示ファイルです。`applyTo` パターンに一致するファイルを編集する際にエージェントへ自動適用されます。

- 成果物種別: `.instructions.md`
- 生成元: 手動作成／Agentによる生成
- 配置先: `.github/instructions/`

| name                                      | 成果物名                   | 概要                                              |
| ----------------------------------------- | -------------------------- | ------------------------------------------------- |
| `markdown.instructions.md`                | Markdown 記述ルール指示    | `docs/**/*.md` に適用する Markdown 記述共通ルール |
| `rules.instructions.md`                   | rules 作成運用ルール指示   | `*-rulebook.md` 作成・更新時の共通運用ルール      |
| `rulebook-to-instruction.instructions.md` | rules→instruction 変換指示 | rules から instruction を生成する際の変換ルール   |
| `rulebook-to-sample.instructions.md`      | rules→sample 変換指示      | rules から sample を生成する際の変換ルール        |

### 2.2. SKILL

複数ステップのエージェントワークフローを定義するスキルファイルです。ユーザーのリクエストに応じてエージェントが呼び出します。

- 成果物種別: `SKILL.md`
- 生成元: 手動作成／Agentによる生成
- 配置先: `.github/skills/<name>/`

| name                      | 成果物名                       | 概要                                                                |
| ------------------------- | ------------------------------ | ------------------------------------------------------------------- |
| `upsert-rulebook`         | upsert-rulebook スキル         | `*-rulebook.md` の新規作成・更新ワークフロー                        |
| `rulebook-to-instruction` | rulebook-to-instruction スキル | rules から instruction を生成するワークフロー                       |
| `rulebook-to-sample`      | rulebook-to-sample スキル      | rules から sample を生成するワークフロー                            |
| `rulebook-lifecycle`      | rulebook-lifecycle スキル      | rules 作成から派生成果物生成までの一連のワークフロー                |
| `rulebook-lifecycle-all`  | rulebook-lifecycle-all スキル  | 全 `*-rulebook.md` を対象にライフサイクルを順次実行するワークフロー |

### 2.3. Prompts

エージェントの特定ワークフローを起動するプロンプトファイルです。チャットから `/` コマンドとして呼び出します。

- 成果物種別: `.prompt.md`
- 生成元: 手動作成／Agentによる生成
- 配置先: `.github/prompts/`

| name                                        | 成果物名                                | 概要                                                       |
| ------------------------------------------- | --------------------------------------- | ---------------------------------------------------------- |
| `upsert-rulebook.prompt.md`                 | rules 新規・更新プロンプト              | `*-rulebook.md` の新規作成・更新を起動                     |
| `upsert-instruction.prompt.md`              | instruction 更新プロンプト              | 指定した `*-instruction.md` の更新を起動                   |
| `upsert-all-instructions.prompt.md`         | 全 instruction 更新プロンプト           | すべての `*-instruction.md` の一括更新を起動               |
| `upsert-changed-instructions.prompt.md`     | 変更 instruction 更新プロンプト         | rules 変更に追従した instruction の更新を起動              |
| `upsert-last-commit-instructions.prompt.md` | 直近コミット instruction 更新プロンプト | 直近コミットの rules 変更に追従した instruction 更新を起動 |
| `upsert-test-instructions.prompt.md`        | テスト instruction 更新プロンプト       | テスト関連 instruction の一括更新を起動                    |
| `upsert-nfr-instructions.prompt.md`         | 非機能要件 instruction 更新プロンプト   | 非機能要件関連 instruction の一括更新を起動                |
| `upsert-migration-instructions.prompt.md`   | 移行 instruction 更新プロンプト         | 移行関連 instruction の一括更新を起動                      |
| `rulebook-lifecycle.prompt.md`              | rules ライフサイクルプロンプト          | rules 作成から派生成果物生成までのライフサイクルを起動     |
| `rulebook-lifecycle-all.prompt.md`          | rules ライフサイクル一括実行プロンプト  | 全 `*-rulebook.md` を対象にライフサイクルを順次実行        |
| `reverse-rulebook.prompt.md`                | 逆引き rules 生成プロンプト             | 既存の成果物ファイルから rules を逆生成する処理を起動      |

## 3. ドキュメント記述ルールと派生物

### 3.1. 成果物種別と派生関係

| 成果物種別  | 生成元                    | 配置先                           | ファイル名              | 説明                                                       |
| ----------- | ------------------------- | -------------------------------- | ----------------------- | ---------------------------------------------------------- |
| rules       | 手動作成／Agentによる生成 | `docs/ja/handbook/rulebooks/`    | `<name>-rulebook.md`    | 各ドキュメントの作成ルール本体（SSOT）                     |
| instruction | rules から派生            | `docs/ja/handbook/instructions/` | `<name>-instruction.md` | rules を生成 AI 向けの実行指示として再構成したテンプレート |
| sample      | rules から派生            | `docs/ja/handbook/samples/`      | `<name>-sample.md`      | rules に準拠した最小の記述例                               |

1 つの rules から 2 つの派生成果物が生成されます。`meta-*-rulebook.md` は派生対象外です。
