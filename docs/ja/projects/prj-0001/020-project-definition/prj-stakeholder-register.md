# ステークホルダー登録簿: SpecDojo Handbook

## 1. 関係者一覧

本書は、SpecDojo Handbook プロジェクトに関係するステークホルダーを識別し、影響度、関心度、必要な合意、関与方針を整理するための登録簿である。

現時点では個人プロジェクトとして開始するため、初期登録対象は、プロジェクトの意思決定、文書作成、AI支援、公開、将来利用に直接関係する最小限の関係者に限定する。

なお、公開リポジトリに配置する文書であるため、個人名、連絡先、非公開の組織情報、機密性の高い利害関係は記載しない。

| ID     | 関係者               | 関与区分                             | 所属/組織                                                        | 役割                                                               | 主な責任                                                     | 備考                                                         |
| ------ | -------------------- | ------------------------------------ | ---------------------------------------------------------------- | ------------------------------------------------------------------ | ------------------------------------------------------------ | ------------------------------------------------------------ |
| SH-001 | プロジェクトオーナー | スポンサー / 意思決定者 / 実行責任者 | SpecDojo Handbook プロジェクト                                   | プロジェクトの目的、方針、優先順位を決定し、成果物を作成・管理する | 目的・範囲・公開方針の決定、文書体系の整備、成果物の最終判断 | 個人プロジェクトとして開始するため、複数の役割を兼ねる       |
| SH-002 | AI Agent             | 実行支援者                           | 利用する生成AI環境                                               | 文書作成、レビュー、構成整理、改善提案を支援する                   | ルールに基づく草案作成、表記揺れ確認、抜け漏れ検出           | 人間の判断や責任を代替しない                                 |
| SH-003 | 将来の利用者         | 利用者                               | 企業、個人、行政、教育機関、地域コミュニティ、OSS プロジェクト等 | SpecDojo Handbook を参照・利用し、仕様駆動開発を実践する           | テンプレート利用、ルール適用、フィードバック提供             | 初期段階では想定ステークホルダーとして扱う                   |
| SH-004 | 将来の貢献者         | 外部協力者 / 新規貢献者              | 個人、企業、OSS コミュニティ等                                   | Issue、Pull Request、提案等を通じて改善に参加する                  | 知見の提供、誤りの修正、テンプレート・ルールの改善           | OSS 公開後に関与が本格化する                                 |
| SH-005 | 配布・公開基盤       | 外部関係者                           | GitHub 等                                                        | Handbook を公開・配布・参照可能にする基盤                          | 公開、履歴管理、Issue/PR、ライセンス表示、参照性の確保       | 実体はサービス基盤であり、管理対象ではなく利用前提として扱う |

## 2. 影響度/関心度分析

影響度と関心度は以下の基準で評価する。

| 評価   | 影響度の基準                                                 | 関心度の基準                         |
| ------ | ------------------------------------------------------------ | ------------------------------------ |
| High   | 方針、スコープ、公開可否、成果物品質、利用拡大に直接影響する | 成果・進行状況に継続的な関与を求める |
| Medium | 一部の成果物、運用、利用、合意形成に影響する                 | 必要時に確認・調整を行う             |
| Low    | 影響は限定的または間接的である                               | 通知・共有のみで足りる               |

| ID     | 関係者               | 関与区分                             | 役割                         | 影響度 | 関心度 | 主な期待                                              | 主な懸念                                             | 必要な合意                                         | 評価根拠                                             |
| ------ | -------------------- | ------------------------------------ | ---------------------------- | ------ | ------ | ----------------------------------------------------- | ---------------------------------------------------- | -------------------------------------------------- | ---------------------------------------------------- |
| SH-001 | プロジェクトオーナー | スポンサー / 意思決定者 / 実行責任者 | 方針決定・文書作成・公開判断 | High   | High   | SpecDojo の目的に沿った実用的な Handbook を整備する   | 方向性のぶれ、過剰な複雑化、継続困難化               | 目的、範囲、公開方針、優先順位、成果物構成         | 最終判断と実行を担うため                             |
| SH-002 | AI Agent             | 実行支援者                           | 作成・レビュー支援           | Medium | High   | ルールに基づき効率的に草案作成・レビューを支援する    | 誤解釈、過剰生成、不整合、根拠のない補完             | 指示テンプレート、入力情報、制約条件、レビュー方針 | 作業効率と品質に影響するが、最終判断は人間が行うため |
| SH-003 | 将来の利用者         | 利用者                               | Handbook の参照・適用        | High   | Medium | 分かりやすく、実務で継続利用できること                | 導入負荷が高い、記述ルールが難しい、適用範囲が不明確 | 適用範囲、利用手順、テンプレート粒度、ライセンス   | 普及と継続利用に直接影響するため                     |
| SH-004 | 将来の貢献者         | 外部協力者 / 新規貢献者              | 改善提案・PR                 | Medium | Medium | 貢献しやすく、改善提案が扱われること                  | 参加方法が不明、レビュー基準が不透明                 | 貢献ルール、Issue/PR 方針、レビュー方針            | OSS としての改善継続に影響するため                   |
| SH-005 | 配布・公開基盤       | 外部関係者                           | 公開・履歴管理・参照基盤     | Medium | Low    | Handbook が参照・取得・改善しやすい形で公開されること | リポジトリ構成が分かりにくい、ライセンスが不明確     | リポジトリ構成、README、LICENSE、公開範囲          | 公開・配布・貢献導線に影響するため                   |

## 4. コミュニケーション要件

本章では、関係者ごとの情報要求、希望チャネル、合意・報告の必要性を整理する。
会議体、報告頻度、配信先、議事録管理等の詳細は、必要に応じてコミュニケーション計画で定義する。

| ID     | 関係者               | 情報要求                                         | 希望チャネル                             | 合意・報告の必要性                        | 証跡要件                         | コミュニケーション計画への反映 |
| ------ | -------------------- | ------------------------------------------------ | ---------------------------------------- | ----------------------------------------- | -------------------------------- | ------------------------------ |
| SH-001 | プロジェクトオーナー | 目的、範囲、優先順位、主要判断事項               | Issue、Pull Request、決定記録            | 方針変更、公開方針、主要成果物の確認      | 決定記録、PR、Issue              | 必須                           |
| SH-002 | AI Agent             | 入力文書、作成ルール、制約条件、出力形式         | 指示テンプレート、チャット               | AI 出力の採否は人間が判断                 | 指示内容、生成結果、レビュー結果 | 必要に応じて反映               |
| SH-003 | 将来の利用者         | 目的、適用範囲、利用手順、テンプレート、サンプル | README、Handbook、導入ガイド             | 利用上の不明点や改善要望を Issue 等で収集 | Issue、フィードバック、更新PR    | 必須                           |
| SH-004 | 将来の貢献者         | 貢献方法、レビュー基準、Issue/PR ルール          | GitHub Issue、Pull Request、CONTRIBUTING | 貢献ルールとレビュー方針の明示            | Issue、PR、レビュー記録          | 必須                           |
| SH-005 | 配布・公開基盤       | リポジトリ構成、ライセンス、更新状況             | GitHub、README、Release notes            | 公開前に README、LICENSE、構成を確認      | README、LICENSE、Git 履歴        | 必須                           |

## 5. 見直し条件

ステークホルダー登録簿は、プロジェクト開始時だけでなく、主要な方針変更、公開準備、外部利用開始、OSS 貢献開始時に見直す。

| 更新トリガー                             | 見直し内容                                             | 見直し責任者         | 承認者               | 証跡                               |
| ---------------------------------------- | ------------------------------------------------------ | -------------------- | -------------------- | ---------------------------------- |
| プロジェクト概要の変更                   | 関係者、影響度、必要な合意の再確認                     | プロジェクトオーナー | プロジェクトオーナー | 変更差分、PR、決定記録             |
| OSS 公開方針の変更                       | ライセンス、公開範囲、将来利用者・貢献者への影響確認   | プロジェクトオーナー | プロジェクトオーナー | LICENSE、README、決定記録          |
| リポジトリ構成の変更                     | 利用者、貢献者、配布基盤への影響確認                   | プロジェクトオーナー | プロジェクトオーナー | PR、Issue                          |
| 主要ルールブックの追加・変更             | 利用者、AI Agent への影響確認                          | プロジェクトオーナー | プロジェクトオーナー | PR、レビュー記録                   |
| 外部からの Issue / Pull Request 受付開始 | 将来の貢献者の関与方針、レビュー方針の確認             | プロジェクトオーナー | プロジェクトオーナー | CONTRIBUTING、Issue、PR            |
| 初回公開前                               | 個人情報、機密情報、ライセンス、README、導入手順の確認 | プロジェクトオーナー | プロジェクトオーナー | 公開前チェックリスト、レビュー記録 |
| 初回公開後の主要フィードバック受領時     | 利用者、貢献者、活用プロジェクトの追加確認             | プロジェクトオーナー | プロジェクトオーナー | Issue、フィードバック、更新PR      |
| 定期見直し                               | 関係者一覧、影響度、関心度、対応方針の更新             | プロジェクトオーナー | プロジェクトオーナー | 定期レビュー記録                   |

定期見直しは、初回公開までは主要な成果物更新時に実施し、初回公開後は少なくとも四半期ごとに実施する。
