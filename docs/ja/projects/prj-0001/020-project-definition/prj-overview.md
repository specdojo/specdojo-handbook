# プロジェクト概要: SpecDojo Handbook

## 1. 背景

生成AIおよびAI Agentの発展により、システムの企画、要件定義、設計、実装、テスト、運用・保守まで、AIが支援・実行できる領域が拡大している。

その一方で、AIが適切に作業するためには、要件、仕様、設計意図、制約条件、判断の経緯などが、曖昧さなく構造的に記述されている必要がある。

従来の開発文書は、主に人間同士の理解と合意形成を目的としていた。しかし今後は、人間に加えてAIも読み手・実行主体となるため、ドキュメントには、人が理解しやすいことに加え、AIが解釈し、再利用し、タスク実行に活用できることが求められる。

本プロジェクトでは、このようなドキュメントを接点として、人とAIが協働しながらシステムを構築・改修・運用し、継続的に改善していく開発スタイルを**仕様駆動開発**と位置付ける。

## 2. 目的

- 仕様駆動開発のための、ライフサイクル全体を支えるオープンなドキュメントフレームワーク **SpecDojo** の **Handbook** を整備する。

- システムの構築・改修・運用・保守に必要な文書群を体系化する。
  記述規約、テンプレート、サンプル、運用ルール、AI向け指示テンプレート、補助ツールを通じて、企画から継続改善までを一貫して支援する。

- 人とAIが共有可能な仕様を中核に据え、開発の生産性、品質、継続性を抜本的に高める。
  これにより、多様なプロジェクトが限られた人材・時間・予算の中でも、社会課題の解消に資するシステムを継続的に構築・改善できる状態を目指す。

- SpecDojo Handbook では、オープンソースとして公開することを中核的な方針とする。
  仕様駆動開発の知見を特定の組織や製品に閉じず、誰もが自由に利用・改善・再利用できる共通基盤として提供する。
  これにより、車輪の再発明を防ぎ、各現場の知見を SpecDojo に還元しやすくし、文書体系や開発プロセスのブラックボックス化を抑制する。

## 3. 必要性

システム開発におけるドキュメンテーションは、担当者の経験や力量に依存しやすく、標準化・再利用・継続的更新が進みにくい。

その結果、多くのプロジェクトでは以下の課題が生じている。

- 要件や設計意図の伝達不足による認識齟齬・手戻り
- 仕様や判断経緯の不足による属人化・ブラックボックス化
- 担当者交代や長期運用時の保守・改修困難化
- AIに与える入力情報の不足による成果物品質のばらつき

特に、社会課題の解消に取り組むプロジェクトでは、限られたリソースの中で、継続的にシステムを改善し続ける必要がある。
そのためには、プロジェクトの知識を人とAIが共有・活用できる形で残し、引き継ぎ、再利用できる基盤が必要である。

SpecDojo は、開発文書を単なる成果物ではなく、人とAIが協働するための共有知識基盤として位置付け、仕様駆動開発に適した文書体系と運用方法を提示する。

## 4. 期待効果

SpecDojo Handbook の整備により、仕様駆動開発を実践可能にし、社会課題の解消に資する課題解決型プロジェクトの成功確率、実行速度、継続性を高めることを期待する。

短期的には、以下の効果を目指す。

- プロジェクトの立ち上げから実装・運用開始までの期間短縮
- 要件、仕様、設計、実装、テストの一貫性向上によるプロジェクト成功率の向上
- AI Agent の活用による、限られた体制での開発実行力向上
- 仕様・設計意図・判断経緯の共有による、継続的な改善・引き継ぎの容易化

中長期的には、以下の効果を目指す。

- 企業、行政、教育、医療、福祉、地域活動、OSS 等における課題解決型プロジェクトの創出数増加
- 社会課題の解消・軽減につながるシステムの実装件数増加
- 小規模組織、非専門家を含むチーム、地域コミュニティによるシステム開発・改善の参加機会拡大
- 価値あるプロジェクトが属人化・資金不足・人材不足により停滞または消滅するリスクの低減

効果は、短期指標と中長期指標に分けて確認する。なお、下記の目標値は、
AI Agent を前提とした仕様駆動開発が十分に機能した場合の挑戦的な初期仮説であり、実測値に基づき継続的に見直す。

## 5. 前提条件

SpecDojo Handbook の策定および活用にあたっては、以下を前提とする。

- SpecDojo は、人間の判断や責任を代替するものではなく、人とAIが協働するための基盤であること
- 対象範囲は、企画、要件定義、設計、実装、テスト、運用・保守、継続改善を含むシステムライフサイクル全体とすること
- ドキュメントは、人間にとっての理解容易性と、AIにとっての解釈容易性を両立すること
- 特定の開発手法、言語、基盤、生成AI製品に過度に依存しないこと
- SpecDojo はオープンソースとして公開し、企業、個人、行政、教育機関、地域コミュニティ、OSS プロジェクト等が自由に利用・改善・再利用できること
- 各現場の知見を SpecDojo に還元しやすい構造とし、継続的に改善される共通基盤を目指すこと
