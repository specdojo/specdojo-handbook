# プロジェクト概要: SpecDojo Handbook の策定

## 1. 背景

- AIの発展によりシステムの企画〜運用・保守のライフサイクルにおいて、Agentが自動でタスクを実行できる領域が広がっている。
- 一方で、Agentが正しくタスクを実行するためには、適切に要件や仕様、設計を伝える必要があり、そのためのドキュメンテーションの重要性が増している
  （以下、これらのドキュメントを接点に、人とAIが協働してシステムを構築・改修する開発スタイルを **仕様駆動開発** と呼ぶ）。
- 仕様駆動開発のためのドキュメンテーションに関しては、様々なプロジェクトや組織で試行錯誤が行われているものの、システム開発に関わるすべての業務プロセスを網羅した体系的なフレームワークは存在しない。

## 2. 目的

- 仕様駆動開発のためのオープンでフルスタックなドキュメントフレームワーク - **SpecDojo** - を整備する。
  - システムの構築・改修に必要なドキュメントを体系化し、記述規約、サンプル、生成AI向け指示テンプレート、
    ツールを通じて、プロダクトのライフサイクル全体を一貫して支援する。
  - AIが扱い易く、人も理解できて、記述も負担にならない、フレームワークを目指す。
- SpecDojo を通じてプロジェクトの生産性と品質を向上させ、様々な社会問題・課題の解決を加速する。

## 3. 必要性

- システム開発の上流工程のドキュメンテーションは職人芸の域にあり、今までも標準的なフレームワークが存在しなかった。
- このために、多くのプロジェクトで失敗やデスマーチが頻発しており、成功したとしても属人化やブラックボックス化が進んでいる。
- 一方で、今まではドキュメントは人が読むためのものであったが、コーディングやテスト、
  運用などのタスクをAIが実行するようになったことで、ドキュメントはAIが読むためのものになった。
- 人間向けからAI向けに、ドキュメンテーションの再定義と体系化が喫緊の課題となっている。
