# トレーサビリティ（要求→仕様）サンプル

## 1. 目的と適用範囲

本書は、駄菓子屋の販売管理システムにおける主要要求と対応仕様の関係を可視化し、要求漏れ・仕様漏れを防ぐためのトレーサビリティを示す。対象は 2026 年度第 1 リリースで扱う会計、在庫、値引き、監査の要求と仕様とする。

## 2. トレース対象の前提

- 要求参照元: 要求一覧 `req-2026-r1`
- 仕様参照元: 業務受入条件 `bac-*`、外部 IF 仕様 `ifx-*`、業務仕様 `bes-*`
- トレース単位: 1 要求に対し 1 つ以上の仕様を対応させる（1:N 許容）
- 対象外: 次期リリース要求（`req-r2-*`）は対象外とする

## 6. 変更履歴と更新ルール

- 要求または仕様の更新が入った場合、同一レビュー単位で本トレースを更新する。
- 充足状況は削除せず、`未着手` / `一部充足` / `充足` の状態遷移で管理する。
- 更新時は備考欄に参照チケット ID を追記する。
