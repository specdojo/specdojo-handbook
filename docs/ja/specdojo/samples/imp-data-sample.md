# 影響調査（データ）サンプル

## 1. 目的と適用範囲

本書は、変更要求 `cr-2026-004` が駄菓子屋の販売管理システムのデータへ与える影響を判断するための文書である。
対象は「商品マスタ」「価格履歴」「売上明細」「キャンペーン適用履歴」の 4 領域とし、判断対象期間は 2026-05-01 から 2026-07-31 とする。

## 2. 変更要求の概要

- 背景: キャンペーン値引きが増加し、集計値と実売上の差異が発生しやすくなっている。
- 要求内容: 値引き適用条件の記録項目追加と、バーコード会計に対応した商品コード運用の見直し。
- 狙い: 集計整合性の確保と、会計処理のデータ品質向上。

## 3. 影響分析サマリ

評価観点は、データ整合性、移行影響、運用負荷の 3 点とする。

| 影響度 | 件数 | 主な対象データ       | 優先対応 |
| ------ | ---- | -------------------- | -------- |
| 高     | 2    | 商品マスタ、売上明細 | 要       |
| 中     | 1    | キャンペーン適用履歴 | 要       |
| 低     | 1    | 価格履歴             | 否       |

## 4. 影響詳細一覧

| 影響対象データ       | 変更要否 | 影響度 | 対応方針 | 判断根拠                                                                             | 備考                         |
| -------------------- | -------- | ------ | -------- | ------------------------------------------------------------------------------------ | ---------------------------- |
| 商品マスタ           | 要       | 高     | 実施     | 商品コード未登録のままではバーコード会計時に照合失敗が発生し、会計停止が起こるため。 | 上位 30 品目を先行整備する。 |
| 売上明細             | 要       | 高     | 実施     | 値引き理由を保持しない場合、日次締め時に差異原因を追跡できず再計算が増えるため。     | 値引き理由コードを追加する。 |
| キャンペーン適用履歴 | 要       | 中     | 代替     | 全項目を即時追加すると運用負荷が高いため、初期は主要条件のみ記録して段階導入する。   | 対象条件を週次で見直す。     |
| 価格履歴             | 否       | 低     | 見送り   | 今回は恒久価格変更ではなく一時値引き対応が中心であり、履歴モデル変更は不要なため。   | 月次監査で再評価する。       |

## 5. 対応方針

| 優先度 | 対応項目                     | 実施時期        | 担当                     | 前提条件                     |
| ------ | ---------------------------- | --------------- | ------------------------ | ---------------------------- |
| 高     | 商品マスタのコード整備       | 2026-05-12 まで | データ管理担当（田中）   | 対象 30 品目の合意           |
| 高     | 売上明細の値引き理由項目追加 | 2026-05-18 まで | 業務システム担当（佐藤） | 集計レポートの項目定義確定   |
| 中     | 適用履歴の段階導入           | 2026-05-31 まで | データ設計担当（山本）   | 運用記録テンプレート作成     |
| 低     | 価格履歴の見送り再評価       | 2026-07-31      | PM（中村）               | 月次監査で差異発生件数を確認 |
