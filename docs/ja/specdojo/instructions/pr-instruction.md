# 進捗報告 作成指示

## 1. 目的と前提

- 目的: 報告期間の進捗、リスク、次アクションを定点観測できる形で記録する。
- 参照ルール: `../rulebooks/pr-rulebook.md`
- 主な内容: 報告期間、完了事項、進行中事項、リスク・課題、次期予定。

## 2. 入力情報

- 報告期間
- 完了事項（成果物IDまたはタスクID）
- 進行中事項
- リスク・課題（影響度、対応方針）
- 次期予定（担当者、期日）

## 3. 出力フォーマット

- Frontmatter は `id` / `type` / `status` を必須とする。
- `id` は `pr-<yyyy-mm-dd>-<nn>`、`type` は `project` を使用する。
- 本文は次の見出し順で作成する。

1. 報告期間
2. 完了事項
3. 進行中事項
4. リスク・課題
5. 次期予定

## 4. 記述ルール

- 完了事項は成果物IDまたはタスクIDと紐づける。
- リスク・課題は影響度と対応方針を明記する。
- 次期予定は担当者と期日を設定する。

## 5. 禁止事項

- 進捗率のみで根拠なく報告しない。
- リスクを抽象語のみで記載しない。
- 予定に期限や責任者を欠いたまま確定しない。

## 6. 最終チェック

- 必須見出しが欠落していない。
- 完了事項に成果物IDまたはタスクIDがある。
- 次期予定に担当者と期日がある。
