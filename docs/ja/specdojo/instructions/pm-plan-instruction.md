# プロジェクト管理計画 作成指示

## 1. 目的と前提

- 目的: プロジェクト全体の管理方針・プロセスを定義する。
- 参照ルール: `../rulebooks/pm-plan-rulebook.md`
- 主な内容: 管理方針、体制、意思決定、レポーティング、見直し条件。

## 2. 入力情報

- 管理対象範囲（スコープ、スケジュール、コスト）
- 品質、リスク、課題の管理方針
- 意思決定体制と権限
- 報告頻度、見直し条件

## 3. 出力フォーマット

- Frontmatter は `id` / `type` / `status` を必須とする。
- `id` は `pm-plan`、`type` は `project` を使用する。
- 本文は次の見出し順で作成する。

1. 概要
2. 管理方針（スコープ/スケジュール/コスト）
3. 品質・リスク・課題管理方針
4. 体制と意思決定
5. レポーティングと見直し
6. 関連ドキュメント

## 4. 記述ルール

- 管理領域ごとに指標、閾値、判断者を明記する。
- 意思決定の権限境界とエスカレーション先を明記する。
- 計画見直しのトリガーと承認プロセスを明記する。

## 5. 禁止事項

- 管理領域を列挙のみで終わらせない。
- 意思決定者や見直し条件を未記載にしない。
- 実装詳細を本文に記載しない。

## 6. 最終チェック

- 必須見出しが欠落していない。
- 管理領域ごとの判断基準が記載されている。
- 関連ドキュメントが空欄でない（`（本書のみ）` 可）。
