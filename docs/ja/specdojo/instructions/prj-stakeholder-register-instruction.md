# ステークホルダー登録簿 作成指示

## 1. 目的と前提

- 目的: プロジェクトの関係者を識別し、影響度・関心度・エンゲージメント方針を可視化して、合意形成の遅延リスクを抑える。
- 参照ルール: `../rulebooks/prj-stakeholder-register-rulebook.md`
- 主な内容: 関係者一覧、影響度/関心度分析、エンゲージメント方針、コミュニケーション要件、見直し条件。
- `prj-overview` を入力として受け取り、`prj-charter` と `pm-organization` の入力として機能する。

## 2. 入力情報

- `prj-overview`（背景・目的・前提条件）
- 関係者候補の一覧（意思決定者、実行者、利用者、外部協力者、影響受容者）
- 関係者ごとの立場・所属・期待・懸念・必要な合意事項
- 対応 Role code の有無（`pm-roles.yaml` または `pm-organization.md` に採用済みのものに限る）
- 未確定事項、プライバシー上の制約（公開可否）

## 3. 出力フォーマット

- Frontmatter に `id` / `type` / `status` / `rulebook` を含める。`id` は `<project-id>-prj-stakeholder-register` を推奨する。
- 本文は次の順序で作成する。

1. 関係者一覧
2. 影響度/関心度分析
3. エンゲージメント方針
4. コミュニケーション要件
5. 見直し条件

- 各章に次の推奨表を配置する。

**関係者一覧:**

| ID  | 関係者 | 関与区分 | 所属/組織 | 対応 Role code | 主な責任 | 備考 |
| --- | ------ | -------- | --------- | -------------- | -------- | ---- |

**影響度/関心度分析:**

| ID  | 関係者 | 関与区分 | 対応 Role code | 影響度 | 関心度 | 主な期待 | 主な懸念 | 必要な合意 | 評価根拠 |
| --- | ------ | -------- | -------------- | ------ | ------ | -------- | -------- | ---------- | -------- |

**エンゲージメント方針:**

| ID  | 関係者 | 現状 | 目標 | 対応方針 | 責任者 | 期限 | 証跡 |
| --- | ------ | ---- | ---- | -------- | ------ | ---- | ---- |

## 4. 記述ルール

- ステークホルダー ID は `STH-<KEY>` 形式（意味付きキー）を使う。例: `STH-SPONSOR`
- 関与区分は固定語彙で記載する（スポンサー / 意思決定者 / 実行責任者 / 実行担当者 / 利用者 / 運用担当者 / 外部協力者 / 影響受容者）。
- 対応 Role code には `pm-roles.yaml` または `pm-organization.md` で採用済みのものだけを使う。Role code が不要な外部集団・基盤には割り当てない。
- ステークホルダーと Role code を混同しない。Role code はプロジェクト内の責務を表し、ステークホルダーはプロジェクト外の関係者も含む。
- 影響度・関心度は High/Medium/Low で評価し、評価根拠（意思決定権限、予算権限、利用影響など）を必ず記載する。
- エンゲージメント方針には責任者・期限・証跡（議事録、Issue、PR、決定記録など）を含める。
- 未確定事項は `_UNDECIDED_`、仮置き前提は `_ASSUMPTION_` で明示する。
- 公開文書では個人名・連絡先情報を記載しない。必要な場合は非公開の管理資料へ分離する。

## 5. 禁止事項

- 対応 Role code に `pm-organization.md` で未採用の Role code を使う。
- 影響度・関心度の評価根拠を省略して主観評価だけにする。
- ステークホルダーを Role code と同一視する（外部関係者への Role code の強制割り当てを含む）。
- 公開文書に個人名、私用メールアドレス、非公開組織情報を記載する。
- 曖昧語（`適切に`、`十分に`）のみで対応方針を記載する。

## 6. 最終チェック

- 5 つの必須章（関係者一覧・影響度/関心度分析・エンゲージメント方針・コミュニケーション要件・見直し条件）がすべて存在する。
- 全ステークホルダーに影響度・関心度・対応方針が設定されている。
- エンゲージメント方針に責任者・期限・証跡が含まれている。
- 対応 Role code がある場合、採用済みの Role code のみを使っている。
- 公開文書として問題のある個人情報が含まれていない。
- 参照ルールの禁止事項に抵触していない。
- Markdown lint でエラーがない。
