# 成果物カタログ索引（DCT Index）作成指示

## 1. 目的と前提

- **目的**: `dct-index.md`（プロジェクト成果物カタログ索引）を 1 ファイル生成する。
- **対象**: プロジェクト単位の `dct-index.md`。ドメイン別の詳細は各 `dct-<domain>.md` に集約し、索引はドメインと参照先リンクの管理に専念する。
- **対象外**: 個別成果物の配置先・根拠・派生関係の詳細（これらは `dct-<domain>.md` に記載する）。

## 2. 入力情報

以下の情報を事前に確認してください。不明な場合は `_UNDECIDED_:` として本文に明記してよい。

- `project-id`（例: `prj-0001`）
- プロジェクトが管理するドメイン一覧（識別子・日本語名・概要）
- 各ドメインの `dct-<domain>.md` の相対パス

## 3. 出力フォーマット

### Frontmatter（必須）

```yaml
---
id: <project-id>:dct-index
type: project
status: draft # draft / ready / deprecated のいずれか
rulebook: dct-index-rulebook
---
```

- `id` は `<project-id>:dct-index` 形式とする（例: `prj-0001:dct-index`）。
- `based_on` は直接根拠がある場合のみ記載する。

### 本文の章構成（必須・順序固定）

| 章  | タイトル           | 必須 |
| --- | ------------------ | ---- |
| 1   | 共通ルール         | ○    |
| 2   | 成果物カタログ一覧 | ○    |

## 4. 記述ルール

### 章 1: 共通ルール

`dct-<domain>.md` 全体に適用される定義を箇条書きで記述する。以下の項目を必ず含める。

| 定義項目                                 | 記述内容                                                                            |
| ---------------------------------------- | ----------------------------------------------------------------------------------- |
| `local-id` の役割                        | 成果物の論理名。ファイル名・Frontmatter `id` の基礎として使用（例: `prj-overview`） |
| ファイル命名規則                         | `<local-id>.md` 形式                                                                |
| Frontmatter `id` 規則                    | `<project-id>:<local-id>` 形式                                                      |
| `根拠` の記載方針                        | 主要な依存関係を `local-id` で記載する                                              |
| `based_on` と `根拠` の関係              | 直接根拠のみ `based_on` に記載し、差分があっても構わない                            |
| 種別定義（`work`/`control`/`generated`） | 種別の定義と「WBS 展開対象は `work` のみ」であることを明示する                      |

### 章 2: 成果物カタログ一覧

- 表形式で記載する。
- 「成果物カタログ」列は `[dct-<domain>](./dct-<domain>.md)` 形式の相対リンクとする。

| 列名           | 説明                                      | 必須 |
| -------------- | ----------------------------------------- | ---- |
| ドメイン       | ドメイン識別子（英小文字・kebab-case）    | ○    |
| 名称           | ドメインの日本語名称                      | ○    |
| 成果物カタログ | `dct-<domain>.md` への相対リンク          | ○    |
| 概要           | ドメイン別カタログの目的・内容を1文で記述 | ○    |

### 命名・ID 運用

- ドメイン識別子は英小文字・kebab-case で統一し、用語集（GL）に沿う。
- `local-id` は再利用しない。廃止時は置き換え関係を明示する。
- プロジェクト内でドメイン識別子の揺れが生じないよう、一覧の識別子を SSOT として使用する。

## 5. 禁止事項

- `dct-index` に個別成果物の詳細情報（配置先・根拠・派生関係など）を記載しない。
- 「概要」列に複数文・長文を記載しない。
- ドメイン識別子に日本語・空白・大文字を使用しない。
- 種別定義を `work` / `control` / `generated` 以外に拡張しない。
- 特定プロジェクト固有の例外事項を共通ルールとして記載しない。

## 6. 最終チェック

- [ ] `id` が `<project-id>:dct-index` 形式になっている
- [ ] 「共通ルール」章に6定義項目がすべて含まれている
- [ ] 「成果物カタログ一覧」表に必須4列がすべて含まれている
- [ ] 「成果物カタログ」列がすべて相対リンク形式になっている
- [ ] ドメイン識別子が英小文字・kebab-case に統一されている
- [ ] 「概要」列が1文以内になっている
