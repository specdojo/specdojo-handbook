# 外部システムI/F一覧 (IFX Index) 作成の指示テンプレート

- 以下のルールに従って、**外部システムI/F一覧（IFX Index）** を 1 つ作成してください。出力は **Markdown** とします。
- 対象は「一覧（表）」であり、実装コードや詳細 API 項目定義は含めません。
- 先頭に YAML Frontmatter を付与し、以下の形式で記述してください（`id` / `type` / `status` は必須）。

  ```yaml
  ---
  id: ifx-index
  type: spec
  status: draft
  rulebook: ifx-index-rulebook
  ---
  ```

- 本文には少なくとも次の見出しを含めてください。
  1. 外部システムI/F一覧
- 一覧表は次の列をこの順序で必ず含めてください。
  - 連携ID
  - 連携名
  - 連携元
  - 連携先
  - 方式
  - フォーマット
  - タイミング
  - 個別仕様
- 「方式」「フォーマット」は語彙を統一し、曖昧語（適宜、その他多数）は使わないでください。
- 個別仕様が存在する場合は、`ifx-api-*` / `ifx-file-*` / `ifx-msg-*` への相対リンクを記載してください。
- 禁止: SQL・クラス名・内部 API 実装詳細・UI クリック手順の記載。
