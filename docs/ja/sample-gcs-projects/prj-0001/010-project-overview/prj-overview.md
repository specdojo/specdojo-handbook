# プロジェクト概要: 駄菓子屋きぬや販売管理システムの構築

## 1. 背景

地域の子どもたちに長年親しまれている駄菓子屋きぬやでは、販売記録や在庫管理をノートと記憶に頼って運用しています。

- 人気商品の欠品が増えてきた
- 仕入れタイミングと数量が勘頼りで在庫のムラが大きい
- 常連のつけ管理が曖昧で記録漏れが起きやすい

## 2. 目的

- 駄菓子の販売、在庫、顧客情報をシンプルに一元管理する
- 店主一人でも無理なく使える画面と操作フローを提供する
- 欠品と過剰在庫を減らし、店舗運営の継続性を高める

## 3. 必要性

- 店主の経験と記憶に依存した運用を続けると、欠品や記録漏れが増えやすい
- 家族が店番するときも同じ手順で販売と在庫確認を行える状態が必要である
- 初期リリースで日々の記録を安定させることが、後続の改善判断の前提になる

## 4. 期待効果

- 人気商品の欠品回数を月間 50% 削減
- 仕入れ金額のムダを年間 20% 削減
- つけ残高を可視化し、会計トラブルを防止

## 5. 前提条件

- 当面は単一店舗で運用し、複数店舗展開は対象外とする
- 店頭で使う端末は 1 台のタブレットを前提とする
- 店舗の運用体制は大きく変えず、現行の販売手順に近い操作を優先する

## 6. 関係者メモ

- 店主が最終的な業務判断を行い、開発リーダーが実現方法と優先順位を整理する
- 具体的なリリース日程と費用上限は別途プロジェクト計画で確定する
