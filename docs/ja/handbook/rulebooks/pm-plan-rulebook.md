# プロジェクト管理計画 作成ルール

Project Management Plan Documentation Rules

本ドキュメントは、`プロジェクト管理計画` を一貫した粒度で作成するためのルールを定義する。

## 1. 全体方針

- プロジェクト全体の管理方針と運用プロセスを定義する。
- 管理領域（スコープ、スケジュール、コスト、品質、リスク、コミュニケーション）を一貫した粒度で記述する。
- 曖昧表現を避け、進行判断に使える記述にする。

## 2. 位置づけと用語定義（必要に応じて）

- `pm-plan`: PM成果物群の中核となる計画書。
- `pm-risk-register`、`pm-issue-log`、`pm-change-request-log`: 監視・統制対象の実績管理。
- `pm-communication-plan`、`pm-organization-and-raci`: 運営体制と連絡計画。

## 3. ファイル命名・ID規則

- `id` は `pm-plan` を使用する。
- ファイル名は `pm-plan-プロジェクト管理計画.md` など一意にする。

## 4. 推奨 Frontmatter 項目

| 項目 | 説明 | 必須 |
| --- | --- | --- |
| id | `pm-plan` | ○ |
| type | `project` 固定 | ○ |
| status | `draft` / `ready` / `deprecated` | ○ |
| based_on | 根拠仕様ID（配列） | 任意 |
| supersedes | 置換関係ID（配列） | 任意 |

## 5. 本文構成（標準テンプレ）

| 番号 | 見出し | 必須 |
| --- | --- | --- |
| 1 | 概要 | ○ |
| 2 | 管理方針（スコープ/スケジュール/コスト） | ○ |
| 3 | 品質・リスク・課題管理方針 | ○ |
| 4 | 体制と意思決定 | ○ |
| 5 | レポーティングと見直し | ○ |
| 6 | 関連ドキュメント | ○ |

## 6. 記述ガイド

### 6.1. 概要

- 計画の目的、適用範囲、計画期間を 1〜3 行で示す。

### 6.2. 管理方針（スコープ/スケジュール/コスト）

- 管理単位、承認基準、逸脱時対応を明記する。

### 6.3. 品質・リスク・課題管理方針

- 品質基準、リスク対応、課題管理の接続を明記する。

### 6.4. 体制と意思決定

- 意思決定体制、権限、エスカレーション先を記載する。

### 6.5. レポーティングと見直し

- 報告頻度、レビュー会議、計画改定条件を記載する。

推奨フォーマット（表）:

| 管理領域 | 指標 | 報告頻度 | 判断者 | 閾値超過時対応 |
| --- | --- | --- | --- | --- |

### 6.6. 関連ドキュメント

- 各PM成果物への導線を記載する。
- 関連がない場合でも `（本書のみ）` を明記する。

## 7. 禁止事項

| 禁止事項 | 理由 |
| --- | --- |
| 管理領域を列挙のみで運用条件を書かない | 実運用に使えないため |
| 意思決定者を未記載にする | 判断遅延が発生するため |
| 見直し条件を未定義にする | 計画が陳腐化するため |
| 実装詳細を本文に記載する | 役割が混在するため |
