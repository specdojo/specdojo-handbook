# WBS 分解戦略 作成ルール

WBS Decomposition Strategy Documentation Rules

本ドキュメントは、`WBS 分解戦略` を一貫した粒度で作成するためのルールを定義する。

## 1. 全体方針

- 成果物カタログを入力として、WBSへ分解する際の粒度・命名・分割方針を定義する。
- 1タスク1責任、完了条件の判定可能性、後続のスケジュール展開可能性を担保する。
- 曖昧表現を避け、分解可否を判断できる記述にする。

## 2. 位置づけと用語定義（必要に応じて）

- `pm-plan`: 全体管理方針。
- `pm-wbs-decomposition-strategy`: 成果物からWBSへ分解する規則。
- `pm-wbs-to-schedule-strategy`: WBSを実行スケジュールへ展開する規則。

## 3. ファイル命名・ID規則

- `id` は `pm-wbs-decomposition-strategy` を使用する。
- ファイル名は `pm-wbs-decomposition-strategy-*.md` など一意にする。

## 6. 記述ガイド

### 6.1. 概要

- 分解戦略の目的、対象範囲、適用期間を 1〜3 行で示す。

### 6.2. 分解原則

- 分解の起点（成果物、要求、機能）と分割単位（機能、工程、担当）を明記する。
- 1タスク1責任、成果物へのトレーサビリティ、重複排除の原則を明記する。

### 6.3. 分解手順

- 入力整理、分解実施、レビュー、確定の順で手順を記載する。
- レビュー観点（漏れ、重複、過分割、依存）を明記する。

### 6.4. 分解粒度と完了条件

- タスク粒度の基準（期間、成果物、責任範囲）を明記する。
- 各タスクに完了条件、受入観点、証跡を定義する。

推奨フォーマット（表）:

| WBS項目 | 親成果物 | 作業内容 | 完了条件 | 責任者 |
| ------- | -------- | -------- | -------- | ------ |

### 6.5. 命名規則とID付番

- WBS項目IDの採番規則、命名規則、改版時ルールを明記する。
- 再利用タスクとプロジェクト固有タスクの区別ルールを明記する。

### 6.6. 関連ドキュメント

- `prj-deliverables-catalog`、`wbs-<domain>`、`pm-wbs-to-schedule-strategy` への導線を記載する。
- 関連がない場合でも `（本書のみ）` を明記する。

## 7. 禁止事項

| 禁止事項                                    | 理由                           |
| ------------------------------------------- | ------------------------------ |
| 分解単位を定義せずにWBSを列挙する           | 再現性が失われるため           |
| 完了条件を未記載のままタスク化する          | 進捗判定できないため           |
| 命名規則・採番規則なしで運用する            | 追跡性が崩れるため             |
| 実装詳細（SQL全文、具体クラス名）を記載する | ドキュメント責務を逸脱するため |
