# コミュニケーション計画 作成ルール

Communication Plan Documentation Rules

本ドキュメントは、`コミュニケーション計画` を一貫した粒度で作成するためのルールを定義する。

## 1. 全体方針

- 報告、連絡、会議体、エスカレーションを計画として明文化する。
- ステークホルダーごとの連絡手段、頻度、責任者を明確にする。
- 曖昧表現を避け、運用時に判断可能な記述にする。

## 2. 位置づけと用語定義（必要に応じて）

- `pm-plan`: 全体管理方針。
- `pm-organization-and-raci`: 体制と責任分担。
- `pm-communication-plan`: 報告・連絡・会議の実行計画。

## 3. ファイル命名・ID規則

- `id` は `pm-communication-plan` を使用する。
- ファイル名は `pm-communication-plan-コミュニケーション計画.md` など一意にする。

## 4. 推奨 Frontmatter 項目

| 項目 | 説明 | 必須 |
| --- | --- | --- |
| id | `pm-communication-plan` | ○ |
| type | `project` 固定 | ○ |
| status | `draft` / `ready` / `deprecated` | ○ |
| based_on | 根拠仕様ID（配列） | 任意 |
| supersedes | 置換関係ID（配列） | 任意 |

## 5. 本文構成（標準テンプレ）

| 番号 | 見出し | 必須 |
| --- | --- | --- |
| 1 | 概要 | ○ |
| 2 | ステークホルダー分類 | ○ |
| 3 | 会議体計画 | ○ |
| 4 | 報告計画 | ○ |
| 5 | 連絡チャネルとエスカレーション | ○ |
| 6 | 関連ドキュメント | ○ |

## 6. 記述ガイド

### 6.1. 概要

- 計画の目的、適用範囲、運用期間を 1〜3 行で示す。

### 6.2. ステークホルダー分類

- 受信者区分、関心事項、必要頻度を記載する。

### 6.3. 会議体計画

- 会議名、目的、参加者、開催頻度、成果物を明記する。

推奨フォーマット（表）:

| 会議名 | 目的 | 参加者 | 頻度 | 成果物 |
| --- | --- | --- | --- | --- |

### 6.4. 報告計画

- 報告種別、提出先、期限、フォーマットを明記する。

### 6.5. 連絡チャネルとエスカレーション

- チャネル、一次連絡先、エスカレーション条件、最終連絡先を明記する。

### 6.6. 関連ドキュメント

- `pm-plan`、`pm-organization-and-raci` への導線を記載する。
- 関連がない場合でも `（本書のみ）` を明記する。

## 7. 禁止事項

| 禁止事項 | 理由 |
| --- | --- |
| 連絡先や責任者を未記載にする | 伝達遅延が発生するため |
| 会議体の目的や成果物を未定義にする | 会議が形骸化するため |
| エスカレーション条件を曖昧にする | 障害時判断が遅れるため |
| 実装詳細を本文に含める | ドキュメント責務から逸脱するため |
