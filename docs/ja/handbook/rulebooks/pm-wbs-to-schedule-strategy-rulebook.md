# WBS からスケジュールへの展開戦略 作成ルール

WBS to Schedule Strategy Documentation Rules

本ドキュメントは、`WBS からスケジュールへの展開戦略` を一貫した粒度で作成するためのルールを定義する。

## 1. 全体方針

- WBS項目を実行可能タスクへ展開するための依存関係、期間見積、進捗更新方針を定義する。
- クリティカルパスの把握、遅延検知、再計画判断が可能な記述にする。
- 曖昧表現を避け、展開可否を判定できる記述にする。

## 2. 位置づけと用語定義（必要に応じて）

- `pm-wbs-decomposition-strategy`: WBS分解の基準。
- `pm-wbs-to-schedule-strategy`: WBSをスケジュールへ変換する規則。
- `sch-<domain>`: 展開後の詳細スケジュール定義。

## 3. ファイル命名・ID規則

- `id` は `pm-wbs-to-schedule-strategy` を使用する。
- ファイル名は `pm-wbs-to-schedule-strategy-*.md` など一意にする。

## 4. 推奨 Frontmatter 項目

| 項目       | 説明                             | 必須 |
| ---------- | -------------------------------- | ---- |
| id         | `pm-wbs-to-schedule-strategy`    | ○    |

## 5. 本文構成（標準テンプレ）

| 番号 | 見出し                 | 必須 |
| ---- | ---------------------- | ---- |
| 1    | 概要                   | ○    |

| 5    | 進捗更新と再計画ルール | ○    |
| 6    | 関連ドキュメント       | ○    |

## 6. 記述ガイド

### 6.1. 概要

- 展開戦略の目的、対象WBS、適用期間を 1〜3 行で示す。

### 6.2. 展開原則

- WBS項目からタスク化する単位、分割条件、統合条件を明記する。
- マイルストーン設定方針、バッファ配置方針を明記する。

### 6.3. 依存関係と順序制約

- `FS` / `SS` など依存種別の使用方針を定義する。
- 外部依存、承認待ち、レビュー待ちの表現ルールを定義する。

### 6.4. 期間・工数見積方針

- 見積手法、前提条件、見積誤差許容を明記する。
- タスク期間の上限・下限、並行実行の条件を明記する。

推奨フォーマット（表）:

| タスクID | 元WBS | 期間見積 | 依存タスク | 進捗判定条件 |
| -------- | ----- | -------- | ---------- | ------------ |

### 6.5. 進捗更新と再計画ルール

- 進捗更新頻度、更新責任者、遅延判定閾値を明記する。
- 再計画トリガー、承認者、記録方法を明記する。

### 6.6. 関連ドキュメント

- `pm-wbs-decomposition-strategy`、`sch-<domain>`、`pm-plan` への導線を記載する。
- 関連がない場合でも `（本書のみ）` を明記する。

## 7. 禁止事項

| 禁止事項                                    | 理由                           |
| ------------------------------------------- | ------------------------------ |
| 依存関係なしで日付のみを並べる              | 実行順が判断できないため       |
| 見積前提を未記載にする                      | 精度と再現性が失われるため     |
| 再計画条件を未定義にする                    | 遅延時判断が遅れるため         |
| 実装詳細（SQL全文、具体クラス名）を記載する | ドキュメント責務を逸脱するため |
