# 成果物カタログ 作成ルール

Deliverables Catalog Documentation Rulebook

本ドキュメントは、プロジェクト成果物の一覧、配置、派生関係を管理するためのルールです。
成果物の漏れ、重複、所在不明を防ぎ、トレーサビリティを担保します。

## 1. 全体方針

- 本ルールの対象は、成果物カタログ（成果物の台帳）です。
- 目的は、成果物の責任者、配置先、依存関係、更新状態を一元管理することです。
- PMBOK 観点では、スコープベースラインと進捗監視に必要な成果物管理情報を含めます。
- 成果物の詳細本文は各成果物本体を正とし、カタログはメタ情報管理に専念します。
- 自動生成成果物と手動作成成果物を区別し、更新責任を明記します。

## 3. ファイル命名・ID規則

### 3.1. 配置（推奨）

- `docs/ja/projects/<project-id>/020-プロジェクトスコープ/` 配下への配置を推奨します。
- 配置先は相対パスで統一し、環境依存の絶対パスを避けます。

### 3.2. ドキュメントID（推奨）

- 推奨: `<project-id>-prj-deliverables-catalog`
  - 例: `prj-0001-prj-deliverables-catalog`

### 3.3. ファイル名（推奨）

- 推奨: `prj-deliverables-catalog.md`
- 日本語ファイル名の場合: `成果物カタログ.md`

### 4.2. 推奨ルール

- 各成果物の責任者と更新トリガーを必ず記載します。
- 派生関係を持つ成果物は、上流成果物 ID を追跡できる形で記述します。

## 5. 本文構成（標準テンプレ）

### 5.1. 成果物カタログ（Deliverables Catalog）

| 番号 | 見出し                 | 必須 | 内容（要点）                   |
| ---- | ---------------------- | ---- | ------------------------------ |
| 1    | 成果物一覧             | ○    | ID、名称、目的、責任者         |
| 2    | 配置と管理方針         | ○    | 配置先、版管理、アクセス方針   |
| 3    | 生成元と派生関係       | ○    | 上流成果物、生成方法、依存関係 |
| 4    | 更新条件と完了条件     | ○    | 更新トリガー、完了判定、証跡   |
| 5    | 未整備成果物と対応計画 | 任意 | 未作成項目、対応期限、担当     |

## 6. 記述ガイド

### 6.1. 共通

- 成果物ごとに「責任者」「更新タイミング」「証跡」を揃えて記載します。
- 章参照は章番号ではなく章タイトルで記述します。
- 廃止済み成果物は削除せず、状態を `deprecated` として残します。

### 6.2. 成果物一覧

- ドキュメント種別（rulebook/instruction/sample/deliverable）を区別します。
- 成果物 ID とファイル名の対応が崩れていないか確認します。

推奨表（成果物一覧）:

| 成果物ID | 名称 | 種別 | 配置先 | 責任者 | 状態 |
| -------- | ---- | ---- | ------ | ------ | ---- |

### 6.3. 生成元と派生関係

- 自動生成物は、生成コマンドまたは生成元を記載します。
- 派生関係の循環依存を禁止します。

推奨表（派生関係）:

| 成果物ID | 生成元 | 生成方法 | 更新トリガー | 証跡 |
| -------- | ------ | -------- | ------------ | ---- |

## 7. 禁止事項

| 項目                 | 理由                         |
| -------------------- | ---------------------------- |
| 成果物IDなしの登録   | 一意識別と追跡ができないため |
| 責任者未記載の成果物 | 更新遅延と放置を招くため     |
| 派生関係の不記載     | 変更影響分析ができないため   |
| 配置先未記載         | 成果物探索コストが増えるため |
