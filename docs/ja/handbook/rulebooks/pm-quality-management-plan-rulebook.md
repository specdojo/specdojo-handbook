# 品質管理計画 作成ルール

Quality Management Plan Documentation Rules

本ドキュメントは、`品質管理計画` を一貫した粒度で作成するためのルールを定義する。

## 1. 全体方針

- 品質目標、レビュー方針、品質基準を明確化する。
- 品質メトリクスの測定方法と判定基準を定義する。
- 曖昧表現を避け、品質判断が可能な記述にする。

## 2. 位置づけと用語定義（必要に応じて）

- `pm-plan`: 全体管理方針。
- `pm-quality-management-plan`: 品質統制の詳細方針。
- `itc-*` / `etc-*` / `stc-*`: 検証計画・検証結果の受け皿。

## 3. ファイル命名・ID規則

- `id` は `pm-quality-management-plan` を使用する。
- ファイル名は `pm-quality-management-plan-品質管理計画.md` など一意にする。

## 4. 推奨 Frontmatter 項目

| 項目 | 説明 | 必須 |
| --- | --- | --- |
| id | `pm-quality-management-plan` | ○ |
| type | `project` 固定 | ○ |
| status | `draft` / `ready` / `deprecated` | ○ |
| based_on | 根拠仕様ID（配列） | 任意 |
| supersedes | 置換関係ID（配列） | 任意 |

## 5. 本文構成（標準テンプレ）

| 番号 | 見出し | 必須 |
| --- | --- | --- |
| 1 | 概要 | ○ |
| 2 | 品質目標 | ○ |
| 3 | レビュープロセス | ○ |
| 4 | 品質メトリクス | ○ |
| 5 | 検査基準と是正 | ○ |
| 6 | 関連ドキュメント | ○ |

## 6. 記述ガイド

### 6.1. 概要

- 品質管理の目的、適用範囲、責任者を 1〜3 行で示す。

### 6.2. 品質目標

- 目標値、達成期限、判定方法を記載する。

### 6.3. レビュープロセス

- レビュー種別、実施者、頻度、出口条件を記載する。

### 6.4. 品質メトリクス

- 指標名、算出方法、閾値、報告先を明記する。

推奨フォーマット（表）:

| 指標 | 算出方法 | 閾値 | 計測頻度 | 報告先 |
| --- | --- | --- | --- | --- |

### 6.5. 検査基準と是正

- 受入基準、不適合時の是正プロセス、再判定条件を明記する。

### 6.6. 関連ドキュメント

- `pm-plan`、`pm-issue-log`、`pm-risk-register` への導線を記載する。
- 関連がない場合でも `（本書のみ）` を明記する。

## 7. 禁止事項

| 禁止事項 | 理由 |
| --- | --- |
| 品質目標を定性のみで記載する | 判定不能になるため |
| メトリクスの算出方法を省略する | 再現性が失われるため |
| 是正プロセスを未定義にする | 不適合が放置されるため |
| 実装依存の詳細を本文に記載する | 役割が混在するため |
