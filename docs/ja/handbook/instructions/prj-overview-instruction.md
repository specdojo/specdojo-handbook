# プロジェクト概要 作成指示

## 1. 目的と前提

- 目的: プロジェクトの背景、目的、必要性、期待効果、前提条件を共有し、意思決定の前提を揃える。
- 参照ルール: `../rulebooks/prj-overview-rulebook.md`
- 主な内容: 背景、目的、必要性、期待効果、前提条件、関係者メモ。

## 2. 入力情報

- プロジェクト名
- 背景（現状、困りごと、発生している問題）
- 目的（達成したい状態）
- 必要性（なぜ今やる必要があるか）
- 期待効果（定量/定性。可能なら指標）
- 前提条件（変えないこと/依存条件/制約）
- 関係者メモ（意思決定者、合意が必要な点、未確定事項）

## 3. 出力フォーマット

- Frontmatter は `id` / `type` / `status` を必須とする。
- `id` は `<project-id>-prj-overview` を推奨し、`type` は `project` を使用する。
- 本文は次の見出し順で作成する。

1. 背景
2. 目的
3. 必要性
4. 期待効果
5. 前提条件
6. 関係者メモ（任意）

## 4. 記述ルール

- 設計詳細（DB、API、画面詳細、テストケース列挙など）は記載しない。
- 背景は「現状」と「問題」を区別して記載する。
- 目的は「改善後の状態」で表現する。
- 期待効果は可能なら「指標/現状/目標/測定方法」をセットで記載し、不明な場合は `未確定` と明記する。
- 前提条件は「変えない前提」と「依存前提」を区別する。

## 5. 禁止事項

- 曖昧語（`適切に`、`可能な限り`、`十分に` など）のみで結論を記載しない。
- 根拠や測定方法がない数値目標を断定しない。
- 参照元不明の値を断定しない。

## 6. 最終チェック

- 必須見出しが欠落していない。
- 期待効果に測定可能な記述、または `未確定` の明示がある。
- 参照ルールの禁止事項に抵触していない。
- Markdown lint でエラーがない。
