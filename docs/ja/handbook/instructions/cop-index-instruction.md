# カットオーバー計画（本番切替手順） 作成指示

## 1. 目的と前提

- 目的: カットオーバー計画の全体SSOT（`cop-index`）を作成し、当日実行の共通判断と導線を定義する
- 参照ルール: `../rulebooks/cop-index-rulebook.md`
- 主な内容: 全体タイムライン、共通GO/NO-GO、共通ロールバック方針、分冊（`cop-<term>`）への導線

## 2. 入力情報

- 対象スコープ、関係者、参照元ドキュメント
- 制約、前提条件、未確定事項

## 3. 出力フォーマット

- Frontmatter は `id` / `type` / `status` を必須とする。
- `id` は `cop-index` を使用する。
- 見出しは連番付きで、本文は `概要（index）` から開始する。

## 4. 記述ルール

- 事実と判断を分離して記述する。
- 当日タイムラインは分単位で記述し、判定点を明記する。
- `cop-<term>` に委譲する詳細手順は重複記載せず、関連ドキュメントで導線を示す。
- 章参照は章タイトルで記述する。

## 5. 禁止事項

- 実装詳細の混入を禁止する。
- 曖昧語のみで結論を記述しない。
- `cop-index` と `cop-<term>` の判断体系が矛盾する記述をしない。

## 6. 最終チェック

- 必須項目が欠落していない。
- 参照ルールの禁止事項に抵触していない。
- Markdown lint でエラーがない。
