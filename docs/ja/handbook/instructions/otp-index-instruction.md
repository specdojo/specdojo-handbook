# 運用切替計画（ハイパーケア含む） 作成指示

## 1. 目的と前提

- 目的: 運用切替計画の入口SSOT（`otp-index`）を作成し、共通判断軸と導線を定義する
- 参照ルール: `../rulebooks/otp-index-rulebook.md`
- 主な内容: 監視/窓口/障害対応/初期増員体制/クローズ条件の共通方針と `otp-<term>` への導線

## 2. 入力情報

- 対象スコープ、関係者、参照元ドキュメント
- 制約、前提条件、未確定事項

## 3. 出力フォーマット

- Frontmatter は `id` / `type` / `status` を必須とする。
- `id` は `otp-index`、`type` は `migration` を使用する。
- 見出しは連番付きで、本文は `概要（index）` から開始する。

## 4. 記述ルール

- 事実と判断を分離して記述する。
- 個別領域の詳細は `otp-<term>` に委譲し、重複記載しない。
- 監視/窓口/SLA/クローズ条件は判定可能な粒度で明記する。
- 章参照は章タイトルで記述する。

## 5. 禁止事項

- 実装詳細の混入を禁止する。
- 曖昧語のみで結論を記述しない。
- `otp-<term>` への導線欠落を禁止する。

## 6. 最終チェック

- 必須項目が欠落していない。
- 参照ルールの禁止事項に抵触していない。
- Markdown lint でエラーがない。
