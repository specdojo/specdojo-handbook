# WBS からスケジュールへの展開戦略 作成指示

## 1. 目的と前提

- 目的: WBS項目を実行可能なスケジュールへ展開する依存関係・期間見積・更新ルールを定義する。
- 参照ルール: `../rulebooks/pm-wbs-to-schedule-strategy-rulebook.md`
- 主な内容: 展開原則、依存関係、見積方針、進捗更新と再計画。

## 2. 入力情報

- WBS一覧（項目ID、内容、完了条件）
- 依存関係情報（先行、後続、外部依存）
- 見積前提（要員、カレンダー、制約）
- 進捗管理ルール（更新頻度、遅延判定閾値）

## 3. 出力フォーマット

- Frontmatter は `id` / `type` / `status` を必須とする。
- `id` は `pm-wbs-to-schedule-strategy`、`type` は `project` を使用する。
- 本文は次の見出し順で作成する。

1. 概要
2. 展開原則
3. 依存関係と順序制約
4. 期間・工数見積方針
5. 進捗更新と再計画ルール
6. 関連ドキュメント

## 4. 記述ルール

- WBSからタスク化する条件とマイルストーン設定方針を明記する。
- 依存種別、外部依存の扱い、順序制約を明記する。
- 見積前提、遅延閾値、再計画トリガーと承認者を明記する。

## 5. 禁止事項

- 依存関係を定義せず日付のみを並べない。
- 見積前提を未記載にしない。
- 再計画条件を未定義にしない。

## 6. 最終チェック

- 必須見出しが欠落していない。
- 依存関係、見積方針、再計画ルールが記載されている。
- 関連ドキュメントが空欄でない（`（本書のみ）` 可）。
