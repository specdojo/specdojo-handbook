# 決定記録 (DEC) 作成の指示テンプレート

- 以下のルールに従って、**決定記録（Decision Log: DEC）** を 1 つ作成してください。出力は **Markdown** とします。
- 対象は `dec-<NNNN>-<topic>` 形式の 1 記録です。
- 先頭に YAML Frontmatter を付与し、以下の形式で記述してください（`id` / `type` / `status` は必須）。

  ```yaml
  ---
  id: dec-<NNNN>-<topic>
  type: project
  status: draft
  ---
  ```

- 本文は次の見出しをこの順序で必ず含めてください。
  1. 背景
  2. 検討した選択肢
  3. 決定内容
  4. 採択理由
  5. 影響範囲
- 「検討した選択肢」は少なくとも 2 案以上を記載してください。
- 「採択理由」は判断軸（品質・コスト・納期・運用性など）を明記してください。
- 「影響範囲」は仕様・実装・運用の 3 観点を含めてください。
- 禁止: 根拠のない結論のみの記載、実装断片のみでの説明代替、影響範囲未記載のまま確定。
