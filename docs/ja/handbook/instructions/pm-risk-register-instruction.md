# リスク登録簿 作成指示

## 1. 目的と前提

- 目的: 識別済みリスクと対応策を管理する。
- 参照ルール: `../rulebooks/pm-risk-register-rulebook.md`
- 主な内容: リスクID、発生確率、影響度、優先度、対応策、担当、状態。

## 2. 入力情報

- リスク一覧（内容、要因、想定影響）
- 評価情報（発生確率、影響度、優先度）
- 対応計画（回避/低減/受容/移転、担当、期限）
- モニタリング情報（トリガー、更新周期）

## 3. 出力フォーマット

- Frontmatter は `id` / `type` / `status` を必須とする。
- `id` は `pm-risk-register`、`type` は `project` を使用する。
- 本文は次の見出し順で作成する。

1. 概要
2. リスク登録簿一覧
3. 評価基準
4. 対応計画
5. モニタリング
6. 関連ドキュメント

## 4. 記述ルール

- リスクごとに一意な `リスクID` を付与する。
- 評価基準を明示し、優先度算定を再現可能にする。
- 対応策には責任者、期限、完了条件を記載する。

## 5. 禁止事項

- リスクIDなしで記録しない。
- 評価基準なしで優先度を決定しない。
- 担当者または対応期限を未記載にしない。

## 6. 最終チェック

- 必須見出しが欠落していない。
- リスクID、優先度、担当、期限が記載されている。
- 関連ドキュメントが空欄でない（`（本書のみ）` 可）。
