# 影響調査（データ）作成の指示テンプレート

Impact Analysis (Data) Instruction Template

このテンプレートは、変更要求がデータへ与える影響を整理する `imp-data` ドキュメントを生成するための指示です。出力は Markdown とし、判断可能な根拠を残してください。

## 1. 目的と前提

- 目的は、変更要求に対するデータ影響を整理し、変更要否・影響度・対応方針・未解決事項を判断可能にすること。
- 実装設計の詳細（SQL、クラス名、API リクエスト構造）は対象外とする。
- 記載粒度は、データ責任者とプロジェクト管理者が合意形成できるレベルとする。

## 2. 入力情報

以下の情報が与えられている前提で作成する。未確定項目は未解決事項として明示する。

- 変更要求 ID と要求概要
- 影響対象のデータ領域、データモデル、データストア
- 変更要否判断に必要な制約（整合性要件、監査要件、移行条件など）
- 影響度評価に使う観点（整合性、可用性、移行難易度、利用者影響など）
- 対応候補（実施、代替、見送り）
- 既存ドキュメントとの関係（必要に応じて `based_on` / `supersedes`）

## 3. 出力フォーマット

### 3.1. Frontmatter

先頭に YAML Frontmatter を置き、以下を満たす。

- 必須: `id`, `type`, `status`, `rulebook`
- 任意: `part_of`, `based_on`, `supersedes`
- `id` は `imp-data` 固定
- `type` は `project`
- `rulebook` は `imp-data-rulebook` を原則とする（該当 rules がない場合のみ `none`）

```yaml
---
id: imp-data
type: project
status: draft
rulebook: imp-data-rulebook
part_of: []
based_on: []
supersedes: []
---
```

### 3.2. 本文見出し

本文は次の見出しをこの順序で含める。

1. 目的と適用範囲
2. 変更要求の概要
3. 影響分析サマリ
4. 影響詳細一覧
5. 対応方針
6. 未解決事項

## 4. 記述ルール

- 影響の有無だけでなく、判断根拠を必ず記載する。
- 影響分析サマリでは、影響度（高・中・低）の件数と優先対応対象を示す。
- 影響詳細一覧は表形式で記載し、以下のカラムを含める。
  - 影響対象データ
  - 変更要否（`要` / `否`）
  - 影響度（`高` / `中` / `低`）
  - 対応方針
  - 判断根拠
  - 備考
- 対応方針には実施時期、担当、前提条件を明記する。
- 見送り項目には再評価条件と再評価時期を必ず記載する。
- 未解決事項には論点、不足情報、決定期限、担当を必ず記載する。

## 5. 禁止事項

- 実装詳細を記載しない。
- 「影響あり」「要対応」のみで根拠を省略しない。
- 影響度の評価尺度を示さずに高・中・低を混在させない。
- 未解決事項を期限や担当なしで残さない。
- 他ドキュメント本文を重複転記し、参照元と不整合な状態にしない。

## 6. 最終チェック

- Frontmatter が `id` / `type` / `status` / `rulebook` を満たしている。
- 本文見出しが指定順序で揃っている。
- 影響詳細一覧の必須カラムが揃っている。
- すべての主要判断に根拠が記載されている。
- 未解決事項に論点・不足情報・決定期限・担当が含まれている。
- 禁止事項に抵触しない。
