# データ移行設計 作成指示

## 1. 目的と前提

- 目的: データ移行設計の入口SSOT（`dmd-index`）を作成し、共通判断軸と導線を定義する
- 参照ルール: `../rulebooks/dmd-index-rulebook.md`
- 主な内容: 共通方針（抽出/変換/除外/冪等性/ログ）、run_id 規約、検証共通方針、対象一覧、`dmd-<term>` への導線

## 2. 入力情報

- 対象スコープ、関係者、参照元ドキュメント
- 制約、前提条件、未確定事項

## 3. 出力フォーマット

- Frontmatter は `id` / `type` / `status` を必須とする。
- `id` は `dmd-index`、`type` は `migration` を使用する。
- 見出しは連番付きで、本文は `概要（index）` から開始する。

## 4. 記述ルール

- 事実と判断を分離して記述する。
- `dmd-<term>` に委譲する個別仕様は重複記載せず、関連ドキュメントで導線を示す。
- 対象一覧では対象/非対象/担当/対応DMDを明記する。
- 章参照は章タイトルで記述する。

## 5. 禁止事項

- 実装詳細の混入を禁止する。
- 曖昧語のみで結論を記述しない。
- run_id 規約または検証共通方針を欠落させない。

## 6. 最終チェック

- 必須項目が欠落していない。
- 参照ルールの禁止事項に抵触していない。
- Markdown lint でエラーがない。
