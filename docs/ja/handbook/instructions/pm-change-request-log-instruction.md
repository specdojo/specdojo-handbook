# 変更要求ログ 作成指示

## 1. 目的と前提

- 目的: 変更要求の申請・審査・決定・実施追跡を管理する。
- 参照ルール: `../rulebooks/pm-change-request-log-rulebook.md`
- 主な内容: 変更ID、要求内容、影響評価、審査結果、決定日、実施状況。

## 2. 入力情報

- 変更要求一覧（要求内容、申請者、申請日）
- 影響評価情報（スコープ、工数、品質、納期）
- 審査結果（承認可否、決定者、決定日、条件）
- 実施追跡情報（担当、期限、進捗、完了条件）

## 3. 出力フォーマット

- Frontmatter は `id` / `type` / `status` を必須とする。
- `id` は `pm-change-request-log`、`type` は `project` を使用する。
- 本文は次の見出し順で作成する。

1. 概要
2. 変更要求一覧
3. 影響評価
4. 審査・決定
5. 実施追跡
6. 関連ドキュメント

## 4. 記述ルール

- 変更要求ごとに一意な `変更ID` を付与する。
- 承認可否には影響評価の根拠を明記する。
- 審査・決定には決定者、決定日、条件を必ず記載する。
- 実施追跡には担当、期限、状態、次アクションを記載する。

## 5. 禁止事項

- 変更IDなしで要求を記録しない。
- 影響評価を省略した承認可否を記載しない。
- 実装コードや詳細差分を本文に貼り付けない。
- 曖昧語のみで結論を記載しない。

## 6. 最終チェック

- 必須見出しが欠落していない。
- 変更ID、決定者、決定日、担当、期限が記載されている。
- 関連ドキュメントが空欄でない（`（本書のみ）` 可）。
