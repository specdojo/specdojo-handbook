# 議事録 作成指示

## 1. 目的と前提

- 目的: 会議の決定事項とアクションを追跡可能に記録する。
- 参照ルール: `../rulebooks/mm-rulebook.md`
- 主な内容: 会議情報、議題、決定事項、アクションアイテム。

## 2. 入力情報

- 会議名、開催日時、参加者
- 議題一覧
- 決定事項（決定者を含む）
- アクションアイテム（担当者、期限を含む）

## 3. 出力フォーマット

- Frontmatter は `id` / `type` / `status` を必須とする。
- `id` は `mm-<yyyy-mm-dd>-<nn>`、`type` は `project` を使用する。
- 本文は次の見出し順で作成する。

1. 会議情報
2. 議題
3. 決定事項
4. アクションアイテム

## 4. 記述ルール

- 決定事項には決定者を明記する。
- アクションには担当者と期限を必ず付与する。
- 保留事項は次回確認日を明記する。

## 5. 禁止事項

- 結論不明の議論のみを羅列しない。
- 担当者未記載のアクションを残さない。
- 実装詳細を本文に過度に記載しない。

## 6. 最終チェック

- 必須見出しが欠落していない。
- 決定事項に決定者が記載されている。
- アクションアイテムに担当者と期限がある。
