# コミュニケーション計画 作成指示

## 1. 目的と前提

- 目的: 報告・連絡・会議体の運営計画を定義する。
- 参照ルール: `../rulebooks/pm-communication-plan-rulebook.md`
- 主な内容: ステークホルダー分類、会議体計画、報告計画、連絡とエスカレーション。

## 2. 入力情報

- ステークホルダー一覧（役割、関心事項）
- 会議体情報（会議名、目的、参加者、頻度）
- 報告情報（報告種別、提出先、期限）
- 連絡チャネルと障害時エスカレーション条件

## 3. 出力フォーマット

- Frontmatter は `id` / `type` / `status` を必須とする。
- `id` は `pm-communication-plan`、`type` は `project` を使用する。
- 本文は次の見出し順で作成する。

1. 概要
2. ステークホルダー分類
3. 会議体計画
4. 報告計画
5. 連絡チャネルとエスカレーション
6. 関連ドキュメント

## 4. 記述ルール

- ステークホルダーごとの受信情報と頻度を明記する。
- 会議体は目的、成果物、責任者を明記する。
- 連絡チャネルには一次連絡先と最終エスカレーション先を明記する。

## 5. 禁止事項

- 連絡先未記載のまま運用計画を確定しない。
- 会議体の目的や成果物を未定義にしない。
- 曖昧語のみで結論を記載しない。

## 6. 最終チェック

- 必須見出しが欠落していない。
- 会議体、報告、エスカレーションの責任者が記載されている。
- 関連ドキュメントが空欄でない（`（本書のみ）` 可）。
