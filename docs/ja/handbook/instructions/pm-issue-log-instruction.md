# 課題ログ 作成指示

## 1. 目的と前提

- 目的: 発生した課題と対応状況を追跡管理する。
- 参照ルール: `../rulebooks/pm-issue-log-rulebook.md`
- 主な内容: 課題ID、内容、優先度、期限、担当、状態、対応策。

## 2. 入力情報

- 課題一覧（発生日、内容、担当）
- 優先度判断材料（影響度、緊急度）
- 対応計画（対応策、期限、依存事項）
- 進捗レビュー情報（更新日、状態遷移）

## 3. 出力フォーマット

- Frontmatter は `id` / `type` / `status` を必須とする。
- `id` は `pm-issue-log`、`type` は `project` を使用する。
- 本文は次の見出し順で作成する。

1. 概要
2. 課題ログ一覧
3. 優先度と期限
4. 対応計画
5. 進捗レビュー
6. 関連ドキュメント

## 4. 記述ルール

- 課題ごとに一意な `課題ID` を付与する。
- `課題ログ一覧` には、課題ID、発生日、内容、優先度、対応策、担当、期限、ステータスを含める。
- 優先度と期限の設定根拠を明記する。
- 状態遷移ルール（未着手/対応中/完了など）を明示する。

## 5. 禁止事項

- 課題IDなしで記録しない。
- 優先度または期限を未記載にしない。
- 根拠のない完了判定を記載しない。

## 6. 最終チェック

- 必須見出しが欠落していない。
- 課題ID、優先度、対応策、期限、担当、状態が記載されている。
- 関連ドキュメントが空欄でない（`（本書のみ）` 可）。
