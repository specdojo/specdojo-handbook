# プロジェクト概要 入力シート

`prj-overview.md` を書き始める前に、最低限の情報を整理するためのシートです。

- **箇条書き・断片・検討中のメモで構いません。** まず書き出すことを優先してください。
- 不明な項目は空欄にせず `未確定` と書いてください（空欄は「まだ考えていない」と区別がつかないため）。
- このシートを埋めたら `prj-overview-rules.md` に従って清書します。

---

## プロジェクト基本情報

| 項目           | 入力値 | 備考                                      |
| -------------- | ------ | ----------------------------------------- |
| プロジェクト名 |        | 例: 駄菓子屋きぬや 店頭販売システムの構築 |
| projectNo      |        | 例: `0001`（IDや配置に使う連番）          |

> 「〜だから困っている」「〜が原因でXXができない」という構造で書くと整理しやすいです。

- 未確認事項:

> **書くべきこと:** このプロジェクトで「何を達成したいか」。完了したときに何が変わっていれば成功か。
> 「〜な状態になる」という形で書くと具体的になります。

## 3. 必要性

> **書くべきこと:** なぜ今やるのか。放置するとどうなるか。期限や外部要因があれば。

（ここに自由に書いてください）

---

## 4. 期待効果

> **書くべきこと:** 達成できたときに何が得られるか。数値で言えるならそれが理想ですが、「未確定」でも可。
> 根拠がない"雰囲気目標"（「30%改善」など）は、測定方法と一緒に書くか「未確定」にしてください。

（ここに自由に書いてください）

---

## 5. 前提条件とスコープ外

> **書くべきこと（前提条件）:** 変えない前提・必要な環境・依存する外部条件。
> **書くべきこと（スコープ外）:** 意図的にやらないことと、その理由。スコープ外を決めると目的が絞れます。

（ここに自由に書いてください）
