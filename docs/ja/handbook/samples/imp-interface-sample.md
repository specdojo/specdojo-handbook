# 影響調査（I/F）サンプル

## 1. 目的と適用範囲

本書は、変更要求 `cr-2026-004` が駄菓子屋の販売管理システムのインターフェースへ与える影響を判断するための文書である。
対象は「POS 連携 API」「商品マスタ連携 CSV」「通知 Webhook」「売上集計バッチ連携」の 4 I/F とする。

## 2. 変更要求の概要

- 背景: バーコード会計とキャンペーン値引き導入により、連携項目の増加が見込まれる。
- 要求内容: 値引き関連項目を連携に追加し、既存連携との互換性を維持する。
- 狙い: 既存連携停止を避けながら、必要な項目を段階的に導入する。

## 3. 影響分析サマリ

評価観点は、契約互換性、障害波及、運用調整負荷の 3 点とする。

| 影響度 | 件数 | 主な対象I/F                  | 優先対応 |
| ------ | ---- | ---------------------------- | -------- |
| 高     | 2    | POS 連携 API、売上集計バッチ | 要       |
| 中     | 1    | 商品マスタ連携 CSV           | 要       |
| 低     | 1    | 通知 Webhook                 | 否       |

## 4. 影響詳細一覧

| 影響対象I/F       | 変更要否 | 影響度 | 対応方針 | 判断根拠                                                                 | 備考                                   |
| ----------------- | -------- | ------ | -------- | ------------------------------------------------------------------------ | -------------------------------------- |
| POS 連携 API      | 要       | 高     | 実施     | 値引き情報を返却しないと会計表示と集計結果に差異が出るため。             | 後方互換のため新項目は任意扱いで追加。 |
| 売上集計バッチ    | 要       | 高     | 実施     | 値引き内訳を取り込まないと日次締めで再集計が必要になるため。             | バッチ失敗時の再実行手順を併記。       |
| 商品マスタ連携CSV | 要       | 中     | 代替     | 全取引先への即時通知は負荷が高いため、先行対象のみ適用して段階拡張する。 | 先行対象は主要取引先 2 社。            |
| 通知 Webhook      | 否       | 低     | 見送り   | 既存通知の必須項目に変更がなく、運用影響が限定的なため。                 | 監視アラート増加時に再評価。           |
